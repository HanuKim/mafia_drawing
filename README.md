# 🕵️‍♂️ 마피아 드로잉 게임 (Mafia Drawing Game)

마피아 드로잉은 실시간으로 다 함께 그림을 그리며 진짜 마피아를 찾아내거나, 마피아가 되어 시민들을 속이는 흥미진진한 웹 기반 멀티플레이어 드로잉 추리 게임입니다.

---

## 🛠 사용 기술 (Tech Stack)

- **Frontend**: HTML5, CSS3, Tailwind CSS (반응형 및 세련된 UI 구성), Vanilla JavaScript (Canvas API를 활용한 드로잉 로직 처리)
- **Backend**: Node.js, Express (정적 에셋 서빙)
- **Communication**: Socket.IO (실시간 양방향 데이터 통신 및 룸/로비 상태 동기화)

---

## 🎮 게임 규칙 (Game Rules)

게임을 시작하려면 최소 **3명 이상**의 플레이어가 필요하며, 한 게임은 총 **3라운드**로 진행됩니다.

### 1. 역할 및 제시어 부여
- 라운드가 시작되면 랜덤으로 1명이 **마피아**, 나머지는 **시민**이 됩니다.
- 시민들에게는 특정 **제시어**(예: 사과, 자동차)가 주어지지만, 마피아에게는 **`?`** 가 표시됩니다.

### 2. 드로잉 페이즈 (그리기)
- 지정된 순서대로 1명씩 **10초씩** 턴을 번갈아 가며 하나의 캔버스에 그림을 그립니다.
- 총 2번의 사이클을 돌며 캔버스에 선을 채웁니다.
- **시민**: 제시어를 설명할 수 있는 정직한 그림을 그립니다.
- **마피아**: 남들이 그리는 그림의 흐름을 눈치껏 파악하며, 정답을 아는 척 자연스럽게 선을 그어야 합니다.

### 3. 투표 페이즈 (15초)
- 모든 드로잉이 끝나면 **15초 동안 투표**가 진행됩니다.
- 캔버스 하단 좌측의 **[그림 분석]** 기능으로 특정 참가자가 그린 선들만 다시 재생해 정밀하게 확인해 볼 수 있습니다.
- 자신이 마피아라고 의심되는 사람을 좌측 참가자 목록에서 클릭하여 투표합니다.
- 과반수 이상의 표를 얻은 유저가 마피아라면, 1차적으로 마피아 검거에 성공합니다!

### 4. 마피아의 최후 반론 (15초)
- 마피아가 검거되었다 하더라도, 마피아에게는 15초의 **최후 반론 기회**가 주어집니다.
- 시민들이 그린 그림을 토대로 마피아가 눈치껏 정답(제시어)을 유추하여 채팅창이 아닌 입력 모달에 정답을 제출합니다.
- 마피아가 **제시어를 맞추면 극적인 마피아의 승리**로 마무리됩니다!

### 5. 라운드 점수 및 명예의 전당 계산
- **마피아 승리**: 마피아 혼자 `+1점` 획득 (검거되지 않고 생존하거나, 검거되었어도 최후 정답을 맞춘 경우)
- **시민 승리**: 마피아를 정확하게 지목(투표)한 시민들에게만 각각 `+1점` 획득
- 3라운드 진행 후, 가장 점수가 높은 **최종 우승자(공동 우승 포함)** 들에게는 **명예의 전당(누적 우승 횟수)이 +1 씩 누적** 기록됩니다!

## ⚙️ 핵심 핵심 로직 및 프로세스 (Logic & Process Flow)

- **글로벌 닉네임 중복 방지 로직**:
  `connectedUsers` Map을 이용해 소켓 접속 시점에 모든 로비와 방을 통틀어 닉네임 중복을 차단합니다. 누군가 브라우저를 끄거나 연결이 끊어지면 `disconnect` 이벤트 내에서 닉네임 점유를 즉시 해지하여 유연성을 제공합니다.
  
- **룸 및 상태 관리 (Room State Machine)**:
  `lobby` -> `drawing` -> `voting` -> `guessing` -> `round_end` -> `game_over` 형태의 엄격한 상태(state) 관리를 서버 단에서 통제합니다. 
  따라서, 네트워크 지연 또는 악의적인 연타에 의해 발생할 수 있는 중복 투표나 중복 점수 상승(Race Condition)을 철저히 방지합니다. (`guess_evaluating` 등 중간 잠금 상태 구현 포함)

- **Canvas 렌더링 동기화 및 기록 분석**:
  마우스나 터치로 인한 드로잉의 `x0`, `y0`, `x1`, `y1`, `color`, `size`, `painterId` 데이터를 소켓을 통해 접속자들에게 즉시 뿌려주어 실시간 선 긋기가 동기화됩니다.
  클라이언트는 수신받은 좌표와 `painterId` 객체들을 `drawingHistory` 배열에 차곡차곡 모아 캐싱해 두며, 투표 시간이 되면 클릭된 버튼에 해당하는 제작자(`painterId`)의 배열 요소들만 캔버스에 리렌더링하여 **개별 그림 보기 기능**을 완벽하게 지원합니다.

- **유동적인 플레이어 이탈 처리**:
  게임 도중 플레이어가 튕기거나 자진해서 방을 나갈 경우, 즉각적으로 인원수(Length)를 체크합니다. 남은 인원수가 진행 최소 요구치(3명)보다 떨어진 즉시 `leave_room` 및 `disconnect` 로직에서 현재 게임의 모든 state 및 round를 무효로 되돌리고 남은 유저들을 대기실로 안전하게 강제 이동합니다.
